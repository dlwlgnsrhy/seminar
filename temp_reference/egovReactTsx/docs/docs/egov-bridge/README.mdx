# eGov 4.3 브리지 체크리스트

목적: 전자정부 요구를 프런트엔드 구현 규칙으로 번역하고, 감리 시 근거를 제시한다.
구성: 각 항목은 “규칙 → 구현 예시 → 감리 체크” 3단으로 작성한다.

---

## A. API 규약/응답 포맷

규칙

- 표준 응답 채택
- 페이지네이션 키 page,size,sort / content,totalElements 고정
- ISO-8601, KST

구현 예시

- shared/api/http.ts, 에러맵, 401 재인증 플로우

감리 체크

- [ ] 응답 모델 문서화
- [ ] 에러 코드 ↔ UX 메시지 테이블 존재
- [ ] 페이지네이션 키 일관

---

## B. 인증/보안

규칙

- HttpOnly 쿠키(+CSRF), 로컬스토리지 금지
- 401 → refresh → 원요청 재시도 → 실패 시 재로그인

구현 예시

- 인터셉터 코드, CSRF 토큰 처리

감리 체크

- [ ] 401 재인증 시나리오 캡처
- [ ] 쿠키/보안 헤더 설정 문서

---

## C. 접근성(KWCAG 2.1 AA)

규칙

- 포커스 링, aria-\*, 대체 텍스트, 대비

구현 예시

- 공통 버튼/모달/폼 접근성 스펙

감리 체크

- [ ] 스크린리더/키보드 내비 동작 캡처

---

## D. 로깅/감사

규칙

- 민감정보 마스킹, 중요 행위 로깅

구현 예시

- 마스킹 유틸, 감사로그 키 연동

감리 체크

- [ ] 삭제/권한 변경 로그 존재

---

## E. 파일 업로드

규칙

- 확장자/MIME 화이트리스트, 크기 제한, 서명 URL 권장

구현 예시

- 업로드 훅/컴포넌트, WebView 권한 안내

감리 체크

- [ ] 대용량/에러 케이스 처리 문서

---

## F. 국제화(i18n)

규칙

- 키 네이밍, 리소스 폴백

구현 예시

- i18n 설정/샘플

감리 체크

- [ ] 한/영 전환 동작 캡처

---

## G. 브라우저 지원/성능

규칙

- 지원 브라우저 명시, 폴리필 정책, 라우트 코드스플릿, 이미지 최적화, 번들 예산

구현 예시

- lazy + Suspense 샘플, 빌드 리포트

감리 체크

- [ ] LCP/바이트 예산 보고

---

## H. 에러 UX

규칙

- 서버/네트워크/권한 에러별 메시지 가이드, 재시도/백오프

구현 예시

- 공통 에러 바운더리/토스트

감리 체크

- [ ] 에러 UX 스크린샷 첨부
