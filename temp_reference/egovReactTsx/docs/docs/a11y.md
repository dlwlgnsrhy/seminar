# 접근성 (A11y) 가이드 - WCAG(Web Content Accessibility Guidelines) 2.1 AA 기준

> **KWCAG 2.1 AA 수준**을 내부 기준으로 채택
> 스크린리더, 키보드 네비게이션, 명도 대비 기준을 모두 반영
> 모든 사용자(시각, 청각, 운동, 인지 등 장애 포함)가 서비스에 동등하게 접근할 수 있도록 한다
> 특히, KWCAG 2.1 AA 레벨을 **개발 단계부터 코드 레벨에서 보장**하는 것을 목표로 한다

---

## 1. 구조적 접근성

- `<main>`, `<nav>`, `<header>`, `<footer>` 시맨틱 태그 사용
- landmark role 중복 금지
- 페이지 타이틀(`document.title`)은 route 별 동적 갱신

## 2. 키보드 네비게이션

- 모든 버튼, 링크는 `tabIndex=0`으로 접근 가능해야 함
- 포커스 트랩 존재 금지 (모달 내에서는 `focusTrap` 허용)
- 포커스 인디케이터(Outline)는 절대 제거 금지

## 3. 색상 및 명도 대비

- 텍스트 대비비율 ≥ **4.5:1**
- 대형 텍스트(≥18pt)는 ≥ **3:1**
- SCSS 변수 기준: `--color-text-primary`, `--color-bg-base`

## 4. 보조기기 지원

- `aria-label`, `aria-labelledby`, `aria-live` 적절히 사용
- 상태 피드백(로딩, 성공, 에러)은 `aria-live="polite"`

## 5. 테스트 도구

- axe-core, Lighthouse A11y tab, Screen Reader (NVDA/VoiceOver)
