# HTML Tags Usage Guide

이 문서는 시멘틱(Semantic) 마크업과 웹 접근성을 고려한 HTML 태그 사용 가이드입니다.

## 1. 시멘틱 태그 (Semantic Tags)

`<div>`만 남발하지 말고, 의미에 맞는 태그를 사용하세요. 이는 SEO(검색 엔진 최적화)와 접근성에 매우 중요합니다.

- `<header>`: 페이지나 섹션의 머리말
- `<nav>`: 네비게이션 링크 그룹
- `<main>`: 문서의 주요 콘텐츠 (페이지당 하나만 존재해야 함)
- `<section>`: 주제별 콘텐츠 그룹 (일반적으로 제목(`h1`~`h6`)을 포함)
- `<article>`: 독립적으로 구분 가능한 콘텐츠 (블로그 포스트, 뉴스 기사 등)
- `<aside>`: 주요 콘텐츠와 간접적으로 관련된 부분 (사이드바, 광고 등)
- `<footer>`: 페이지나 섹션의 꼬리말

## 2. 텍스트 관련 태그

- `<h1>` ~ `<h6>`: 제목 태그. `<h1>`은 페이지당 하나만 사용하는 것이 원칙입니다. 순서를 건너뛰지 마세요 (h1 -> h3 (X)).
- `<p>`: 문단(Paragraph).
- `<ul>`, `<ol>`, `<li>`: 목록. 순서가 없으면 `ul`, 있으면 `ol`을 사용합니다.
- `<button>`: 클릭 가능한 동작을 수행할 때 사용합니다. (링크 이동은 `<a>` 사용)
- `<a>`: 다른 페이지나 URL로 이동할 때 사용합니다.

## 3. 폼 관련 태그

- `<form>`: 사용자 입력을 서버로 전송하는 양식.
- `<label>`: 입력 필드의 라벨. `for` 속성(React에서는 `htmlFor`)으로 `input`의 `id`와 연결해야 합니다.
- `<input>`: 다양한 타입의 입력 필드 (`text`, `checkbox`, `radio` 등).
- `<select>`, `<option>`: 드롭다운 목록.
- `<textarea>`: 여러 줄 텍스트 입력.

### 접근성 팁

모든 입력 필드에는 연결된 `<label>`이 있거나, `aria-label` 속성을 제공해야 스크린 리더 사용자가 인식할 수 있습니다.

```tsx
// Good
<label htmlFor="username">Username</label>
<input id="username" type="text" />

// Good (숨겨진 라벨)
<input type="text" aria-label="Search" />
```

## 4. 이미지 태그

- `<img>`: 이미지 삽입. `alt` 속성은 **필수**입니다. 이미지가 로드되지 않거나 스크린 리더 사용자를 위해 이미지에 대한 설명을 적어야 합니다.

```tsx
<img src="logo.png" alt="Company Logo" />
```

- 장식용 이미지라면 `alt=""`로 비워두세요.

## 5. 자주 하는 실수

- **버튼 vs 링크**:
  - 페이지를 이동한다면 `<a>` (React Router의 `<Link>`)
  - 액션을 수행한다면(제출, 모달 열기 등) `<button>`
- **블록 요소 안에 블록 요소**:
  - `<p>` 태그 안에는 `<div>`를 넣을 수 없습니다.
  - `<a>` 태그 안에는 인터랙티브 요소(`<button>`, `<input>`)를 넣으면 안 됩니다.
